Sizing intelligence layer

Understand how sizes differ across brands.

FitMatch is shifting from a body-measurement quiz to a structured sizing dataset and translation layer. The goal is simple: compare brands, normalize size charts, and make sizing data useful before we ever ask a customer for body input.

Try size translation Inspect dataset

Product direction

  • Brand fit comparison tool
  • Size translation engine
  • Normalization layer for public size charts
  • Future feedback loop from users

Body-based recommendation can come later, but it should sit on top of this layer, not replace it.

What changed from the previous MVP

The old version centered on a small questionnaire and body measurements for women's T-shirts. This iteration moves the core object from the person to the sizing system.

Before: "What size fits this person?"
Now: "How do sizes differ across brands and products?"
Later: "What can user feedback teach the dataset?"
